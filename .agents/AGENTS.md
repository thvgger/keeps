# Project Rules

## Skeleton Loaders
- Always use skeleton loaders instead of plain text loading states (e.g. "Loading...").
- Skeleton loaders should mirror the layout they're replacing — match the same structure, spacing, and element shapes so the transition from loading to loaded feels seamless.
- Use `skeleton-dark` for elements on dark backgrounds and `skeleton-line` for elements on light/colored backgrounds (see `globals.css`).
- Skeleton cards should use the actual card background colors at reduced opacity to hint at the real content.
- Inner skeleton bars should have varied widths to look natural, not uniform.

## Code Comments
- Never add comments (inline comments, block comments, or explanatory comments) to the code unless explicitly requested. Keep code clean and self-explanatory.
